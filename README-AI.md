# Архитектура Видеозвонков (Глубокий Аудит + E2EE Phase 5+)

Этот документ содержит полное описание архитектуры системы "Secure Relay" и всех оптимизаций, внедренных для обеспечения полной приватности через E2EE.

## 1. Общая концепция
Проект реализует нестандартную архитектуру со стримингом сырых медиаданных поверх WebSocket. Стандартные возможности WebRTC (RTP/VP8/H264) намеренно не используются для обеспечения обхода DPI и гибкого контроля над трафиком.

## 2. Локальный AI (BPS/Adaptive)
Центральным узлом управления является `AdaptiveVideoEngine` (ранее `AdaptiveJPEGEngine.ts`). Он реализует механизм контроля задержек на базе **WebCodecs API**:
*   **Истинная пропускная способность ($R_{hat}$):** 
    Алгоритм вычисляет реальную скорость дренажа сети по формуле: 
    $R_{hat} = \frac{\Delta BytesSent - \Delta BufferedAmount}{\Delta t}$. 
*   **FSM (Автомат состояний):**
    - `probe`: Поиск максимума канала.
    - `steady`: Стабильный режим.
    - `congested`: Экстренный сброс до $0.85 \times R_{hat}$. 
    - `recovery`: Быстрый выход из "ямы" (+8%/сек).
*   **iOS Safari Fix (iosBufferPanic):** Сброс очередей при RTT > 1500мс или RTT > 5x Baseline.

## 3. Транспортный слой и Pacer
*   **Token Bucket Pacer:** Пакеты распределяются по времени (smoothing), предотвращая Bufferbloat.
*   **AES-GCM Encryption:** Все полезные нагрузки (payloads) зашифрованы MANDATORY.
*   **V2 Packet Markers:**
    - `1`: Raw PCM (отключено при E2EE)
    - `3`: Encrypted PCM Audio
    - `0xFF marker`: Encrypted H.264 Annex B Frame Slice

## 4. Безопасность и E2EE (Mandatory)
Система реализует полноценное шифрование "из конца в конец".

*   **ECDH Key Exchange (P-256):**
    Генерация эфемерных пар ключей Elliptic Curve Diffie-Hellman. На обоих концах вычисляется **Shared Secret**.
*   **AES-256-GCM Payload Encryption:**
    - Использование аппаратного ускорения Web Crypto API.
    - Для каждого пакета видео и аудио генерируется уникальный случайный 12-байтовый **IV**.
*   **Key Lifecycle:** Ключи живут только в оперативной памяти и уничтожаются при вызове `cleanup()`, устанавливая `sharedSecretRef.current = null`.
*   **Mandatory Mode:** Если `publicKey` отсутствует в сигнальном событии `call_user` / `answer_call` или дешифровка провалилась — звонок прерывается.

## 5. Видео и Аудио
*   **WebCodecs (VideoEncoder):** Видео кодируется аппаратно (AVC1 Annex B, Level 3.1) для поддержки разрешения до 1280x720 и минимальной задержки.
*   **PCM 8-bit:** Аудио захватывается в 8-битном формате для экономии 50% трафика.
*   **Catch-up Logic:** В `MJPEGDecoder` реализован сброс буфера при накоплении задержки > 150 мс.

---
**Security Note:** Релейный сервер не имеет доступа к ключам и видит только зашифрованные пакеты.
